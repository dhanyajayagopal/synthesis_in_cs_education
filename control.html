Binary Search Trees

Introduction to Binary Trees

A binary tree is a tree in which each node has exactly two children, either of which may be empty. For example, the following is a binary tree:

Note that some of the nodes above are drawn with only one child or no children at all. In these cases, one or both children are empty. Note that we always draw one child to the left and one child to the right. As a result, if one child is empty, we can always tell which child is empty and which child is not. We call the two children the left child and the right child. We can implement a single node of a binary tree as a data structure and use it to store data. The implementation is as follows:

class Node: def __init__(self, val): self.l_child = None self.r_child = None self.data = val

In a binary tree, the only invariant is that each node has a value, a left child, and a right child.

Introduction to Binary Search Trees (BSTs)

In a binary search tree (BST), there is an additional invariant. A binary search tree is ordered, so there are three factors that must hold:

  1. All values in the tree are unique.
  2. Each node value is greater than all values in its left child.
  3. Each node value is less than all values in its right child.

Why are binary search trees important?

The reason binary search trees are important is that the following operations can be implemented efficiently using a BST:

  • Insert a value into the tree.
  • Determine whether a value is in the tree.
  • Remove a value from the tree.
